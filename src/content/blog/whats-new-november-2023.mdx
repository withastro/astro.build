## Releases

Even with 4.0 just around the corner, this month saw several Astro releases!

- [Astro 3.5](https://github.com/withastro/astro/releases/tag/astro%403.5.0): Integrations can now add other integrations, added `<form>` support and Prefetch. You can also try out the experimental i18n!
- [Astro 3.6](https://github.com/withastro/astro/releases/tag/astro%403.6.0): This release added three new events to view transitions.
- [Starlight 0.12](https://github.com/withastro/starlight/releases/tag/%40astrojs%2Fstarlight%400.12.0): Override Starlight‚Äôs UI with custom components and a lot of improvements.
- [Starlight 0.13](https://github.com/withastro/starlight/releases/tag/%40astrojs%2Fstarlight%400.13.0): Built-in support for [Expressive Code](https://github.com/hippotastic/expressive-code) code block features.
- [Starlight 0.14](https://github.com/withastro/starlight/releases/tag/%40astrojs%2Fstarlight%400.14.0): Adds a new [plugin API](https://starlight.astro.build/reference/plugins/) and launches official support for [Algolia DocSearch](https://starlight.astro.build/guides/site-search/).

## Docs

The Docs team is working hard to migrate the Astro Documentation to Starlight! We have very good news for you! It‚Äôs coming next week!

To help you better contribute, we've added `help - confirm behaviour` and `help - leave feedback` to our list of PR labels to clarify where we need help. Open Issues and PRs have been freshly triaged and there's lots to dive into. 

Our weekly Discord call "Talking and Doc‚Äôing" is a chance for you to watch Team Docs work in public, jump in a codespace, and contribute to the docs. This week's topics included:

- Test driving the new Astro Docs in Starlight
- Reviewing and merging the completed French tutorial
- Rebuilding the [astro.build](http://astro.build) footer so we could link our new community website
- Testing a new Starlight `<Testimonial />` component
- Building the Astro v3 to v4 upgrade guide
