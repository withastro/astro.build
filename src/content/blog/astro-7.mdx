## Upgrade now

To upgrade an existing project to Astro 7, use the automated `@astrojs/upgrade` CLI tool:

```sh
# Recommended:
npx @astrojs/upgrade

# Manual:
npm install astro@latest
```

For new projects, simply use:

```sh
npm create astro@latest
```

### Rust Compiler

{/*
  - Graduated from experimental (introduced in Astro 6) to stable default
  - Successor to the Go-based .astro compiler
  - Faster, stronger diagnostics, more reliable
  - Now the default — no experimental flag needed
  - @astrojs/compiler-rs is no longer a separate install
  - Link: AST-136
*/}

### Sätteri: Markdown & MDX in Rust

{/*
  - Sätteri is now the default Markdown processor (replaces unified/remark/rehype)
  - Introduced in Astro 6.4 as an opt-in processor via markdown.processor config
  - Much faster than unified pipeline — shaved over a minute off Astro and Cloudflare docs builds
  - Implements many Markdown features natively that previously required plugins
  - unified() still available via @astrojs/markdown-remark for projects that depend on remark/rehype plugins
  - Deprecated remark/rehype top-level config options removed (deprecated in 6.4)
  - Link: AST-162, AST-141
  - RFC: https://github.com/withastro/roadmap/pull/1364
*/}

### Queued Rendering

{/*
  - Graduated from experimental (introduced in Astro 6) to stable default
  - Replaces recursive rendering with a two-pass approach: traverse + ordered queue, then render
  - Faster and more memory-efficient
  - Early benchmarks showed up to 2x faster rendering
  - Now the default rendering strategy — no flag needed
  - Link: AST-166
*/}
