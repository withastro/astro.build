## Upgrade now

To upgrade an existing project to Astro 7, use the automated `@astrojs/upgrade` CLI tool:

```sh
# Recommended:
npx @astrojs/upgrade

# Manual:
npm install astro@latest
```

For new projects, simply use:

```sh
npm create astro@latest
```

### Queued Rendering

Queued rendering is stable, and the default rendering engine. The recursive approach has been replaced with a queued-based \
approach where nodes are visited, pushed into a queue, and rendered.

```js title="astro.config.mjs" del={5-9}
import { defineConfig } from "astro/config";

export default defineConfig({
	experimental: {
		queuedRendering: {
			enabled: true,
			pooling: true,
			contentCache: 1000
		}
	}
})
```

The experimental implementation used a two-pass approach:
1. Create an ordered list of components (nodes).
2. Loop the list and render the components.

The new implementation doesn't create a full list anymore, instead the list is rendered (flushed) while it's looped. This final approach is faster than the first iteration, and needs less memory compared to the recursive approach. Early benchmarks showed that it's **~2.4× faster**[^1] in speed.

The node pooling technique has been removed because it didn't yield enough savings, while making the code less maintainable.

The content cache technique has been greatly descoped. The Astro renderer now caches only the values generated by HTML tags.

[^1]: The savings are visible in expressions-dense pages. Static pages wouldn't see much savings.

### JSON Logging

Custom logging is now stable, and users can switch to structured logging from CLI or configuration:

```shell ins={2} del={1}
astro dev --experiemntalJson
astro dev --json
```

```js title="astro.config.mjs" del={4,6}
import { defineConfig, logHandlers } from "astro/config";

export default defineConfig({
	experimental: {
		logger: logHandlers.json()
	}
})
```

Additionally, when running the development server with the `--background` flag, Astro automatically uses to JSON logging.
