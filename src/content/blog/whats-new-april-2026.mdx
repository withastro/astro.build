The latest community-built [utilities and integrations](/integrations/) to help you build with Astro. 🛠

- [@fujocoded/astro-authproto](https://github.com/FujoWebDev/fujocoded-plugins/tree/main/astro-authproto): Adds ATproto authentication to your Astro site. Allows your visitors to log into your website using their account on any ATproto services (like Bluesky).
- [@jdevalk/astro-seo-graph](https://github.com/jdevalk/seo-graph/tree/main/packages/astro-seo-graph#readme): Astro integration with SEO component, route factories, content-collection aggregator, Zod content helpers.
- [@markdy/astro](https://github.com/HoangYell/markdy-com): Astro island component for writing MarkdyScript animations to generate animated scenes directly in the browser.
- [@flagify/astro](https://flagify.dev/docs/v1/sdk/astro/): An Astro integration for evaluating feature flags in pages and components, with a dev toolbar for overrides.
- [bodegacat](https://www.npmjs.com/package/bodegacat): Astro integration for Bodega Cat e-commerce stores (Cloudflare + Stripe).
- [astro-woocommerce](https://github.com/Ardesh1r/astro-woocommerce.git): Type-safe WooCommerce integration for Astro. Fetch products, categories, and manage cart state with full TypeScript support.
- [@hedystia/astro](https://www.npmjs.com/package/@hedystia/astro): Use @hedystia/view components within Astro.
- [@zdenekkurecka/astro-consent](https://github.com/zdenekkurecka/astro-consent#readme): Astro integration for GDPR/ePrivacy-friendly cookie consent: banner, preferences modal, runtime API, category-based consent state, strict-CSP safe, works with or without View Transitions.
- [astro-magic-move](https://github.com/cabljac/astro-magic-move): Animated code morphing for Astro, powered by Shiki Magic Move. Build-time tokenization, zero-framework client JS, CSS-variable theming.
- [@ahmttyydn/astro](https://www.npmjs.com/package/@ahmttyydn/astro): Scalar API Reference Component for Astro. Render a beautiful API reference based on an OpenAPI/Swagger document. 
- [astro-pigment](https://github.com/psd-coder/astro-pigment#readme): An Astro 6 documentation theme with dark mode, interactive playgrounds, and SEO endpoints.
- [@casoon/astro-sitemap](https://github.com/casoon/astro-sitemap#readme): Astro integration that generates sitemap.xml at build time — supports dynamic sources, pattern-based rules, i18n hreflang, serialize hook, and sitemap-index chunking.
- [@twodft/astro-icon](https://github.com/twodft/astro-icon#readme): A community-maintained fork of `astro-icon` by Nate Moore, updated for Astro v6.x and Cloudflare Workers compatibility.
- [@omniaura/astro-grab](https://github.com/omniaura/astro-grab): Hold a key, hover, click — grab any element's source context for AI agents. Astro devtool.
- [@casoon/astro-crawler-policy](https://github.com/casoon/astro-crawler-policy#readme): Policy-first crawler control for Astro — generates robots.txt and llms.txt with presets, per-bot rules, AI crawler registry, and build-time audits.
- [astro-bangumi](https://www.npmjs.com/package/astro-bangumi): A Bangumi integration for Astro, similar to `hexo-bilibili-bangumi`.
- [astro-include-asset](https://github.com/SamuNatsu/astro-include-asset): Include assets on your page by writing a custom remark directive in your Markdown/MDX file.
- [@pliosoft/sites-sdk](https://github.com/pliosoft/sites-sdk#readme): Shared Astro components, layouts, and design tokens for Pliosoft-hosted customer sites.
- [astro-cloudflare-analytics](https://github.com/JamiesonRoberts/astro-cloudflare-analytics/#readme): An Astro component to handle manual Cloudflare Analytics installation.
- [astro-research-publications](https://www.npmjs.com/package/astro-research-publications): Astro integration that syncs your academic publications from Google Scholar, Semantic Scholar, OpenAlex, and/or ORCID and renders a publications page.
- [astro-llms-md](https://github.com/tfmurad/astro-llms-md#readme): Generate llms.txt, llms-full.txt, and individual markdown files from your Astro site with this Astro integration.
- [@red-glare/astro](https://github.com/IHIutch/red-glare): Astro integration for building USWDS-styled federal documentation sites with Comark.
- [@soufianeh/annotate-code](https://www.npmjs.com/package/@soufianeh/annotate-code): Link code lines to explanations — an Astro integration and remark plugin with hover highlighting.
- [@jessgaspardev/astro-seo](https://github.com/jessgaspardev/astro-seo#readme): A fully-featured SEO component for Astro with Open Graph, Twitter Cards, JSON-LD and more.
- [@dotslide/framework](https://github.com/Hanziness/dotSlide): A modern presentation framework for Astro. Build fast, portable decks from ready-made semantic components instead of custom slide code.
- [astro-omni-compress](https://github.com/dharanish-v/omni-compress/tree/master/packages/astro-omni-compress#readme): A drop-in alternative to sharp for Astro's `<Image>` component that works wherever sharp's native binaries fail — Docker alpine images, serverless functions, Vercel Edge, Cloudflare Workers.
- [@at-flux/astro-feature-flags](https://github.com/at-flux/astroflare/tree/main/packages/astro-feature-flags): Typed Astro feature flags from JSON config. Includes environment specific rendering and pruning from elements to entire routes.
- [@twodft/astro-favicons](https://github.com/twodft/astro-favicons#readme): An all-in-one favicon and PWA assets generator for Astro projects. It automates the creation of favicons, manifest, and supports hot reloading for efficient development.
- [@tsparticles/astro](https://github.com/tsparticles/tsparticles#readme): Easily create highly customizable particle, confetti and fireworks animations with ready to use components and use them as animated backgrounds.
- [@obaronai/astro-ai-readiness](https://github.com/obaronai/astro-ai-readiness#readme): AI Readiness toolkit for Astro.
- [astro-tinylytics](https://github.com/tylerbutler/tools-monorepo/tree/main/packages/astro-tinylytics#astro-tinylytics): Astro components for Tinylytics analytics, kudos buttons, and webmentions.
- [astro-meta-engine](https://github.com/TheElegantCoding/astro-meta-engine#readme): Powerful, type-safe metadata and JSON-LD engine for Astro.
- [astro-gtm-lite](https://github.com/tfmurad/astro-gtm-lite#readme): Lightweight Google Tag Manager compatible with Astro View Transitions (client router).
- [@mintlify/astro](https://github.com/mintlify/astro) - Syncs Mintlify-style docs content into Astro content collections and provides MDX components for rendering docs pages.
- [astro-dgmo](https://github.com/diagrammo/astro-dgmo#readme): Astro integration to render DGMO diagrams from fenced code blocks at build time.
- [astro-opengraph](https://github.com/altano/npm-packages/tree/main/packages/astro-opengraph) - An integration to create and manage your Open Graph images, including a dev toolbar app to preview the image and the `og` meta tags on the page.
- [@ayco/astro-resume](https://git.ayo.run/ayo/astro-resume): Utilities for serializing data from the server for use in the client.
- [@better-translate/astro](https://github.com/jralvarenga/better-translate/tree/main/packages/astro): A type-safe i18n toolkit for TypeScript.
- [astlide](https://github.com/r-hashi01/astlide): An Astro-based slide presentation framework — like Slidev, but for the Astro ecosystem.
- [astro-feedkit](https://github.com/kitschpatrol/astro-feed-kit): An Astro integration for full-content RSS, Atom, and JSON feeds.
- [astro-mdx](https://github.com/kitschpatrol/astro-mdx-kit): An Astro integration for MDX directive-to-component mapping, element overrides, and auto-imports.

### Plugins

- [Astro Cursor Plugin](https://cursor.directory/plugins/astro-1): Cursor plugin for Astro including the official Astro Docs MCP, rules with best practises and coding standards and several skills such as add integration, content collection, create component, docs lookup and migrate.
- [ESLint astro/no-prerender-export-outside-pages](https://ota-meshi.github.io/eslint-plugin-astro/rules/no-prerender-export-outside-pages/): An `eslint-plugin-astro` rule for Astro that warns when `export const prerender` is used outside the `pages/` directory.

### CMS/text editors
