## Latest releases

Major releases introduce breaking changes, stabilize features, and provide improvements. Minor releases introduce new features and improvements. Check out what's new!

- [Astro 6 is here](/blog/astro-6/) with a refactored dev server, an experimental Rust compiler, live content collections, CSP, and more.
- [Astro 6.1](/blog/astro-610/) - More control over image optimization, more markdown configuration options, updated hooks for working with i18n, and more.

// Integrations added as of `"created": "2026-03-23`

- [Backstro Email](https://github.com/backstrojs/email) Compose email templates with Astro components — a direct port of react-email component to native `.astro` syntax.
- [Review Loop](https://github.com/viv/review-loop) - A dev-only annotation overlay for reviewing rendered sites and feeding corrections to AI coding agents.
- [Astro Techradar](https://github.com/dirsigler/astro-techradar) - An interactive, fully static technology radar built with Astro. Track technology adoption decisions with a visual radar chart — driven entirely by Markdown files.
- [NPM Astro Render Component](https://github.com/igorskyflyer/npm-astro-render-component) - Astro component renderer. Zero configuration. Produces clean HTML strings directly in Node.js without any DOM environment.
- [Lexington SEO](https://github.com/Lexington-Themes/lexington-seo) - Render SEO tags in your page `<head>` using a single AstroSeo component.
- [Astro Markdown Export](https://github.com/louisbrulenaudet/astro-markdown-export) - An Astro integration that exports content-collection Markdown files at build time and enhances their frontmatter with optional source URLs and extra metadata for bots, crawlers, AI agents, and other tools.
- [Astro Reveal](https://github.com/polgubau/astro-reveal) - Beautiful scroll-reveal animations for Astro with zero JavaScript in production. GPU-accelerated, performant, and accessible.
- [Neokit](https://github.com/ppreeper/neokit) - An Astro component library that automatically injects Tailwind + DaisyUI into your project — no manual CSS imports needed.
- [Astro Bundle Budget](https://github.com/shiftEscape/astro-integrations/tree/main/packages/astro-bundle-budget#readme) - Build-time JS/CSS bundle size budgets for Astro. Inspect every asset and page payload at the end of `astro build` — and optionally fail CI when you exceed your thresholds.
- [LCARS Stylus](https://github.com/withstudiocms/lcars-stylus) - LCARS inspired design system built with Stylus.
- [Astro Icon Sprite](https://github.com/Livog/astro-icon-sprite) - Build-time SVG compilation into a runtime sprite sheet.
- [Astro Stacks](https://github.com/Livog/astro-stacks) - A Laravel Blade-style `@stack` / `@push` / `@pushOnce` system for Astro SSR.
- [Astro MD Alternate](https://github.com/gxjansen/astro-md-alternate) - An Astro integration to serve markdown versions of content pages for AI agents and LLMs.
- [Astro Securitytxt](https://github.com/dirsigler/astro-securitytxt) - An Astro integration that automatically generates an RFC 9116 compliant `security.txt` file during your site build.
- [Astrobit](https://github.com/SouichiroTsujimoto/astrobit) - A thin framework for running MoonBit components as Astro islands.
- [JAAMD](https://github.com/lancher-dev/jaamd) - Just Another Astro Markdown — remark plugins, client-side enhancements and styles as a single Astro integration.
- [Agent Markup](https://github.com/agentmarkup/agentmarkup) - Build-time llms.txt, JSON-LD, markdown mirrors, AI crawler controls, and validation.
- [CoCart Headless SDK](https://github.com/cocart-headless/cocart-js) - Build headless WooCommerce storefronts through its API instead of using PHP templates.
- [GreenLight VSCode extension](https://marketplace.visualstudio.com/items?itemName=GreenLight.greenlight-vs) - A visual drag-and-drop HTML editor for Markdown, HTML, and Astro components.
