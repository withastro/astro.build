## Releases

Astro's development keeps marching forward, with several notable releases in October:

- [Astro 3.2](/blog/astro-320/): This update gives you more control over the browser history stack, enables page navigation with View Transitions, includes a route announcer for screen readers, and integrations can now dynamically add other integrations.
- [Astro 3.3](/blog/astro-330/): This release introduces an experimental `<Picture />` component, enhances syntax highlighting, brings package provenance, and adds various quality-of-life improvements.
- [Astro 3.4](/blog/astro-340/): Introducing Page Partials, along with improvements in image optimization and a first preview of an experimental dev overlay.
- [Starlight 0.11](https://github.com/withastro/starlight/releases/tag/%40astrojs%2Fstarlight%400.11.0): Adds support for customizing Starlight‚Äôs UI with component overrides, adding attributes to sidebar links, and improves `<title>` tags.

See the `#announcements` channel in the [Astro Discord](https://astro.build/chat) for releases across all our repos, including Starlight, the VS Code extension, compiler, packages, and integrations! Follow [Astro on GitHub](https://github.com/withastro/) for all updates.

## Docs

We extended the tutorial with some highly requested features!

- [Content Collections](https://docs.astro.build/en/tutorials/add-content-collections/)
- [View Transitions](https://docs.astro.build/en/tutorials/add-view-transitions/)

"Talking and Doc'ing" is our weekly live Discord call where we involve the community as we work on docs projects! Often we jump into a shared Codespace or HackMD document and everyone can participate. This month's activities included:

- Live reviewing and testing the two new mini-tutorials
- Improving our docs PR template and creating a new GitHub action to welcome first-time contributors
- All hands on deck reviewing translation submissions using the Astro Docs checklist to get everyone's `hacktoberfest-accepted` PRs merged
