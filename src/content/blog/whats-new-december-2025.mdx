## Tips & Tools

The latest community-built [utilities and integrations](/integrations/) to help you build with Astro. üõ†

- [Astro Auto Load](https://www.npmjs.com/package/astro-auto-load) - Automatic component-level data loading for Astro SSR.
- [Astro Content Viewer](https://github.com/dofbi/astro-content-viewer) - A Dev Toolbar app for Astro that visualizes your content collections, Zod schemas, and associated variables.
- [Astro Dynamic Component](https://github.com/shjyh/astro-dynamic-component) - An Astro integration that enables dynamic component imports with glob patterns and client directives support.
- [Astro Shortlinks](https://github.com/Ovyerus/astro-shortlinks) - Automatically create shortlinks for your Astro site.
- [Astro Tips](https://github.com/xingwangzhe/astro-tips) - Beautiful, customizable tip and notice blocks for Astro with 18 built-in styles and auto dark mode support.
- [Astro Compress](https://github.com/jgabriel98/astro-compress) - A powerful compression integration for static assets in Astro.
- üåü [Starlight GitHub Stats](https://www.npmjs.com/package/starlight-github-stats) - A Starlight plugin to display GitHub repository statistics (stars, forks, and latest version) in the header of your documentation site.
- [Simple Tips](https://www.npmjs.com/package/super-tips) - A simple and easy-to-use Astro integration for creating beautiful tooltips in Markdown and MDX files.
- [Astro Speed Measure](https://github.com/casoon/astro-speed-measure) - Measure Astro build performance end-to-end. Wraps integrations and Vite plugins, aggregates per-page and asset timings, and emits human, JSON, or HTML reports.
- [EdgeOne Astro Adapter](https://github.com/TencentEdgeOne/astro-adapter) - An adapter that allows you to deploy your Astro site to EdgeOne Pages.
- [Astro Pirsch Proxy](https://github.com/freshcodes/astro-pirsch-proxy) - An Astro integration that exposes a Pirsch analytics proxy.
- [Auto Param Astro](https://github.com/newgentdigital/auto-param-astro) - Astro integration that adds configured query parameters to external links at build time.
- [Astro SEO](https://github.com/ByManuelCaballero/astro-seo) - SEO component for Astro with full TypeScript support.
- [Astro Deck](https://github.com/sandikodev/astro-deck) - Command deck for Astro - Admin hub with CMS, AI content generator, and developer tools.
- [sweetcorn](https://github.com/delucis/sweetcorn) - JavaScript image dithering tools for Sharp. Turn smooth pixels into crunchy kernels.
- [Astro Content AI Translator](https://github.com/affiliatefm/astro-content-ai-translator) - Astro integration for AI-powered content translation using OpenAI.
- [CMS Marker](https://github.com/nuasite/nuasite/blob/main/packages/cms-marker/) - An Astro integration that automatically marks HTML elements with unique identifiers and generates a manifest mapping content to source files and line numbers. Perfect for building CMS editors that need to track where content originates in your codebase.
- [Astro Click to Source](https://github.com/invisible1988/astro-click-to-source) - An Astro integration for Alt+Click to open source files in your editor during development.
- [ez i18n](https://github.com/zachhandley/ez-i18n) - Cookie-based i18n for Astro + Vue + React. No URL prefixes, reactive language switching.
- [Astro Marp](https://github.com/astro-marp/astro-marp) - An Astro integration that transforms `.marp` Markdown slide sources into optimized presentation pages with full asset pipeline integration and server-side Mermaid diagram rendering.
- [Astro NocoDB](https://github.com/dofbi/astro-nocodb) - An Astro loader package that integrates NocoDB data into Astro content collections.
- [Spectre UI Astro](https://github.com/phcdevworks/spectre-ui-astro) - First‚Äëclass Astro layer for Spectre UI. Typed, SSR‚Äëfriendly Astro components (Button, Card, Input and more) that wrap Spectre UI‚Äôs classes and tokens.
- [Astro Shield](https://github.com/kindspells/astro-shield) - An Astro integration to enhance your website's security with SubResource Integrity hashes, Content-Security-Policy headers, and other techniques.
- [Writenex Astro](https://github.com/erlandv/writenex/tree/main/packages/astro) - Visual editor for Astro content collections - WYSIWYG editing for your Astro site.
- [Dynamic Storage Interface](https://github.com/Adammatthiesen/s3-interface-test) - Reusable Astro components for cloud storage file management. Currently implements S3-compatible storage providers (AWS S3, Backblaze B2, MinIO, etc.).
- [Bejamas UI](https://github.com/bejamas/ui) - Astro-Native Building Blocks for Modern Design Systems.
- [Vault CMS](https://github.com/davidvkimball/vault-cms) - Use Obsidian as a content management system for your Astro website.
- [Pageel](https://github.com/pageel/pageel-core) - A powerful, client-side Content Management System (CMS) for Astro and Next.js for managing Markdown/MDX content and images directly on GitHub, Gitea, or Gogs repositories.
